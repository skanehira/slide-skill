# SLIDE-PATTERN: org-chart-tree

## Overview

最上位ノード（役員会等）から部門、役職・業務リストへと階層的に分岐するツリー型組織図。垂直線と水平線で接続する。

## 適したシーン

会社・部署・プロジェクトの組織体制、意思決定構造、チームの役割分担を階層的に示すとき。

## Structure

```yaml
structure:
  layout: tree
  levels: 3
  level_1: [board-node1, board-node2]  # 最上位
  level_2: [dept1, dept2, dept3, dept4]  # 部門
  level_3: per-dept role-list  # 役職リスト
  connectors: vertical-horizontal-lines
```

## Elements

| 要素 | 配置 | 役割 |
|---|---|---|
| 最上位ノード | 第1行・上部 | 取締役会・監査役会等の最高意思決定機関 |
| 部門ノード | 第2行・中部 | CEO直下・各事業部門のノード（背景色付き） |
| 役職・業務リスト | 第3行・下部 | 各部門に所属する役職・業務名の縦リスト |
| 接続線 | ノード間 | 垂直線（親子）・水平線（兄弟）で階層を表現 |
| ロゴ | 左下 | 会社ロゴ・クレジット（省略可） |

## Usage Guide

- 第2行の部門数は3〜5が適切。6以上は横幅が狭くなる
- 各部門の役職リストは4〜6項目が最適
- 最上位ノードは2つ並べる場合（取締役会＋監査役会）が多い
- 背景色付きノードはブランドカラーを使用
