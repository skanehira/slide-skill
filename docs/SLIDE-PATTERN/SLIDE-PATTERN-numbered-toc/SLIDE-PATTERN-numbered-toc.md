# SLIDE-PATTERN-numbered-toc

このファイルはスライドのコンテンツエリア（タイトル行より下の領域）のレイアウトパターン定義書です。SLIDE.mdと組み合わせてAIツールに渡すことで、このパターンのスライドを生成できます。タイトルエリア・ページ番号・装飾はSLIDE.mdの `Slide Frame` セクションで定義されるため、このファイルには含みません。

## Overview

**パターン名：** numbered-toc
**概要：** 番号・項目名・点線・ページ番号を横一列に並べたリスト行を縦積みで表示する目次レイアウト
**適したシーン：** プレゼンテーション・提案書の目次スライド、章立てが明確なドキュメントの構成提示

## Structure（構造）

コンテンツエリア全体に縦積みのリストを中央揃えで配置する。各行は「番号. 項目名」「点線（ドットリーダー）」「ページ番号」の3要素が横並びになっており、行間に十分な余白を持たせて視認性を高める。

    structure:
      layout: single
      alignment: center-left
      rows:
        type: numbered-list-item
        elements: [number, item-name, dotted-leader, page-number]
        count: 4〜8行（項目数に応じて調整）
        spacing: 行間に余白を設けてゆったりと表示

## Elements（各要素の役割）

※ タイトル（H1）はSLIDE.mdの Slide Frame で定義されるため、この表には含めない。コンテンツエリアの要素のみを記述する。

| 要素 | 配置 | 役割 |
|---|---|---|
| 番号 | 各行・左端 | 目次項目の連番（「1.」「2.」形式） |
| 項目名 | 各行・番号の右 | 章・セクションの名称（日本語または英語） |
| 点線（ドットリーダー） | 各行・項目名とページ番号の間 | 項目名とページ番号を視覚的に繋ぐ装飾線 |
| ページ番号 | 各行・右端 | 該当セクションの開始ページ番号 |

## Usage Guide（AIへの使い方）

このパターンをAIに指示する際のプロンプト例：

> 「SLIDE-PATTERN-numbered-tocのレイアウトで、[項目名とページ番号のリスト]を使って目次スライドを作成してください。デザインはSLIDE.mdに従ってください。」

**注意点：**
- 点線はCSSの `border-bottom: dotted` またはテキストの「------」で代替できる
- 番号と項目名の間のスペースを揃えることで整列感が出る
- 項目数が多い場合（8行以上）はフォントサイズを小さくするか、2カラム構成に変更することを検討する
- 現在のセクションを強調色で表示するバリエーションも有効
